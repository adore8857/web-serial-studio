# Web Serial Studio - 数据解析与自定义协议指南

在 Web Serial Studio 中，系统提供了多种数据解析模式，以适配不同类型的硬件设备和数据流格式。本文档详细解释了这些模式的区别，并特别说明了如何通过修改 JSON 配置文件来适配将来硬件协议的变动。

## 1. Frame Parsing (帧解析模式) 详解

左侧边栏（Sidebar）提供了以下几种标准的帧解析模式：

### 1.1 Quick Plot Mode (快速绘图模式)
- **适用场景**：最简单、最通用的串口打印格式。
- **数据格式**：硬件直接通过 `printf` 等函数发送纯文本字符串，每个数据之间用逗号（`,`）隔开，每一帧以换行符（`\n` 或 `\r\n`）结束。
- **示例数据**：`25.4, 55.1, 1013.2, 9.81\n`
- **UI 表现**：系统会自动抓取这些数字，并在仪表盘上直接绘制对应的折线图，**无需任何前置配置**。

### 1.2 Device Sends JSON (设备自述模式)
- **适用场景**：智能化硬件，硬件自身知道自己有哪些传感器，并能自己定义 UI 布局。
- **工作规则**：
  1. 硬件连接后，首先向网页发送一段特定格式的 JSON 字符串（通常以特殊的定界符包裹，如 `/* ... */`），用于描述界面需要几个 Plot、几个 Gauge、数据叫什么名字、单位是什么。
  2. 随后，硬件开始持续发送像 `Quick Plot` 那样的逗号分隔数据。
- **优势**：即插即用，彻底免去在网页端进行任何设置。

### 1.3 Project File (工程文件模式) - **推荐模式**
- **适用场景**：固定设备的监控面板，或者需要展示极其复杂的二进制数据流（例如我们的 STM32 高频轴承数据）。
- **工作规则**：
  1. 用户事先在网页的 Editor 中拖拽生成，或者手动编写好一个 JSON 布局配置文件。
  2. 点击 `📂 Open Project` 加载该配置文件。
  3. 系统会根据文件中的布局生成美观的 UI，并按照配置将接收到的数据流（CSV 或特定的 Binary 协议）提取并映射到 UI 控件上。
- **特例**：在这个模式下，如果 JSON 配置文件指定了特定的二进制协议（如 `"protocol": "STM32Binary"`），系统会自动切换到底层的高性能二进制解包引擎，忽略默认的文本解析。

## 2. STM32 自定义二进制协议开发指南

为了达到最高的传输效率，你的 STM32 系统使用的是**紧凑的二进制帧传输**（无逗号，无换行），而非文本传输。

当你未来修改了 STM32 的传输协议（例如增加了一个传感器，或者改变了发送顺序），你需要进行以下**两步操作**来适配网页端：

### 步骤一：修改底层解析引擎 (`src/core/FrameParser.js`)

由于二进制解包高度依赖于字节偏移量（Offset）和端序（Endianness），这部分目前直接硬编码在解析引擎中以获得最大性能。

打开 `src/core/FrameParser.js`，找到 `_parseSTM32Binary()` 函数：

```javascript
// 示例：从偏移量 12 处提取 2132 个 16 位小端整数
const vib = [];
for (let i = 0; i < 2132; i++) {
  vib.push(view.getInt16(12 + i * 2, true)); 
}

// 示例：将提取出的数据放入 dataset
this._emitFrame({
  title: 'STM32 Bearing Data',
  datasets: [
    { title: 'Vibration', value: vib[vib.length - 1], index: 0, buffer: vib },
    // index: 0 是这个数据的全局唯一标识，它将和 JSON 配置文件对应
  ]
});
```

**修改规则：**
- 如果你增加了数据，只需使用 `view.getInt16()`、`view.getFloat32()` 或者自定义的 24 位函数从正确的偏移量提取数据。
- 给这个新数据分配一个不重复的 `index`（例如 `index: 6`）。

### 步骤二：编写/修改 JSON 布局文件

解析引擎负责提取数据，而 JSON 文件负责告诉系统“把哪些数据画在哪里”。

打开你的项目文件（如 `stm32-bearing.json`），它的核心结构如下：

```json
{
  "title": "STM32 Bearing Monitoring System",
  "protocol": "STM32Binary",  // 关键！这一行告诉系统不要用普通的 CSV 解析，而是使用上面修改的二进制引擎
  "groups": [
    {
      "title": "新传感器监控区",
      "widget": "Plot",       // 控件类型：Plot, MultiPlot, Gauges, Bar 等
      "datasets": [
        { 
          "title": "新数据名称", 
          "index": 6,         // 必须和 FrameParser.js 里分配的 index 一致！
          "min": -10, 
          "max": 10, 
          "units": "mA" 
        }
      ]
    }
  ]
}
```

当你修改完毕并保存后，只需在 Web 界面重新加载这个 JSON 文件，新的界面布局和数据流通道就会自动建立连接。

> **附注：关于图表的局部放大与平移**
> 在目前的 UI 中，我已为绘图控件（PlotWidget）集成了高级的缩放交互功能：
> - **缩放 (Zoom)**：直接在折线图上使用鼠标滚轮上下滚动，即可沿 X 轴（时间轴）局部放大或缩小密集的数据点。
> - **平移 (Pan)**：按住 `Shift` 键，然后使用鼠标左键拖拽图表区，即可左右平移查看历史或未来的数据曲线细节。
